## 一、零基础快速上手

### 1.1 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS、Linux 均可 |
| Python | 3.9 ~ 3.11（推荐 3.10） |
| 内存 | 8GB 以上 |
| 网络 | 需要联网（调用大模型 API） |

### 1.2 安装步骤（5 分钟）

**第一步：打开终端**

- Windows：按 `Win + R`，输入 `cmd`，回车
- macOS：按 `Cmd + 空格`，输入 `终端`，回车

**第二步：进入项目目录**

```bash
cd "C:\Users\lenovo\Desktop\English OCR"
```

**第三步：安装依赖**

```bash
pip install -r requirements.txt
```

> ⏳ 首次安装约需 5-10 分钟，总共下载约 2GB（EasyOCR 模型 + PyTorch 等）

**第四步：构建知识库**

```bash
python knowledge/build_kb.py
```

> ✅ 看到 "Knowledge base built: 254 entries" 即表示成功
>
> 💡 如果网络不通（无法访问 HuggingFace），系统会自动切换到**关键词匹配模式**，不影响使用

**第五步：启动系统**

```bash
python app.py
```

浏览器会自动打开，或手动访问 **http://localhost:7860**

## 二、配置 API Key（必做）

系统需要调用大模型进行翻译和分析，**必须配置 API Key**。

### 2.1 选择一家大模型提供商

| 提供商 | 获取地址 | 免费额度 | 推荐理由 |
|--------|---------|---------|---------|
| **DeepSeek (推荐)** | [platform.deepseek.com](https://platform.deepseek.com) | 注册送 500 万 tokens | 性价比最高，中文强 |
| 通义千问 | [dashscope.aliyun.com](https://dashscope.aliyun.com) | 100万 tokens/月 | 中文翻译强 |
| 智谱 GLM | [open.bigmodel.cn](https://open.bigmodel.cn) | 100万 tokens/月 | 性价比不错 |
| OpenAI | [platform.openai.com](https://platform.openai.com) | 注册送 $5 | 综合最强但贵 |

### 2.2 获取 API Key 流程（以 DeepSeek 为例，推荐）

1. 访问 [platform.deepseek.com](https://platform.deepseek.com)
2. 用手机号注册（国内手机直接注册）
3. 点击左侧「API Keys」→「创建 API Key」
4. 复制保存 Key（格式：`sk-xxxxxxxxxxxxxxxx`）
5. 新用户注册送 **500 万 tokens**，足够作业使用

### 2.3 配置到系统（二选一）

**方式一：环境变量（推荐，更安全）**

```bash
# Windows CMD
set LLM_API_KEY=sk-xxxxxxxxxxxxxxxx

# Windows PowerShell
$env:LLM_API_KEY="sk-xxxxxxxxxxxxxxxx"

# macOS / Linux
export LLM_API_KEY="sk-xxxxxxxxxxxxxxxx"
```

**方式二：直接写配置文件**

打开 `config.py`，找到第 23 行，修改为：
```python
LLM_API_KEY = "sk-xxxxxxxxxxxxxxxx"
```

### 2.4 切换大模型提供商

打开 `config.py`，修改第 47 行：
```python
DEFAULT_PROVIDER = "deepseek"   # 可选: deepseek, deepseek-r1, qwen, glm, openai
```

> 💡 **推荐用 DeepSeek**：国内直接访问，注册简单，送 500 万 tokens，API 兼容 OpenAI 格式。

## 三、离线安装指南（外源复制模型）

如果网络无法下载模型，可以从其他电脑或外源复制已下载的模型文件到对应位置。

### 3.1 EasyOCR 模型放置位置

EasyOCR 的识别模型存放在用户目录下：

| 系统 | 路径 |
|------|------|
| **Windows** | `C:\Users\你的用户名\.EasyOCR\model\` |
| **macOS** | `/Users/你的用户名/.EasyOCR/model/` |
| **Linux** | `/home/你的用户名/.EasyOCR/model/` |

**需要的文件**（英文手写识别模型 `english_g2`）：
```
.EasyOCR/
└── model/
    └── english_g2.pth          # 英文识别模型 (~68MB)
```

**操作步骤**：
1. 从外源复制 `english_g2.pth` 文件
2. 在用户目录下创建 `.EasyOCR\model\` 文件夹
3. 将 `english_g2.pth` 放入该文件夹
4. 系统启动时会自动检测本地模型，不再下载

### 3.2 PyTorch 安装

PyTorch 是 **pip 包**（不是模型文件），需要安装到 Python 环境中：

**在线安装（推荐）**：
```bash
# CPU 版本（无 NVIDIA 显卡时，~150MB）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu

# GPU 版本（有 NVIDIA 显卡时，~2.5GB）
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

**离线安装（从外源复制 .whl 文件）**：
1. 在能上网的电脑上，从 [PyTorch 官网](https://pytorch.org/get-started/locally/) 或镜像站下载对应的 `.whl` 文件
2. 复制到本机
3. 手动安装：
```bash
pip install "torch-2.x.x-cp312-cp312-win_amd64.whl"
```

> ⚠️ 注意：`.whl` 文件名必须匹配你的 Python 版本（如 `cp312` = Python 3.12）

### 3.3 Sentence-Transformers 模型放置位置

知识库的语义嵌入模型（可选，不用也能正常工作）：

| 系统 | 路径 |
|------|------|
| Windows | `C:\Users\你的用户名\.cache\huggingface\hub\` |
| macOS | `/Users/你的用户名/.cache/huggingface/hub/` |
| Linux | `/home/你的用户名/.cache/huggingface/hub/` |

**需要的文件**（`all-MiniLM-L6-v2` 模型，~80MB）：
```
.cache/huggingface/hub/
└── models--sentence-transformers--all-MiniLM-L6-v2/
    └── snapshots/
        └── <hash>/
            ├── config.json
            ├── model.safetensors
            ├── tokenizer.json
            ├── vocab.txt
            └── modules.json
```

**操作步骤**：
1. 在能访问 HuggingFace 的电脑上执行：
   ```bash
   python -c "from sentence_transformers import SentenceTransformer; SentenceTransformer('all-MiniLM-L6-v2')"
   ```
2. 下载完成后，复制整个 `.cache/huggingface/hub/` 文件夹
3. 粘贴到本机的相同路径

> 💡 **不用这个模型也可以**：系统会自动使用关键词匹配模式，功能正常。详见第 3.4 节。

### 3.4 各模型的作用与必要性

| 模型 | 大小 | 必须？ | 作用 | 没有的话…… |
|------|------|--------|------|------------|
| EasyOCR `english_g2.pth` | ~68MB | ✅ **必须** | OCR 手写识别 | 系统无法识别图片 |
| PyTorch | ~150MB~2.5GB | ✅ **必须** | EasyOCR 底层依赖 | OCR 无法运行 |
| Sentence-Transformers | ~80MB | ❌ 可选 | 知识库语义检索 | 自动切换关键词匹配 |
| LLM API | 云端 | ✅ **必须** | 翻译/纠错/分析 | 分析功能不可用 |

### 3.2 操作流程

**步骤 1：上传图片**
- 点击左侧「上传」区域，选择手写英文图片
- 或直接把图片拖进去
- 支持 JPG、PNG、BMP 等常见格式
- 也可以用截图工具（微信/QQ 截图）截图后 Ctrl+V 粘贴

**步骤 2：自动 OCR 识别**
- 图片上传后，系统自动进行预处理和 OCR 识别
- 识别到的英文文本出现在中间栏
- 你可以直接编辑识别文本（修正 OCR 错误后重新分析）

**步骤 3：一键分析**
- 点击「🚀 一键分析」按钮
- 系统执行：知识库检索 → 大模型多维分析
- 等待 10-30 秒（取决于网络和 API 响应速度）
- 结果出现在右侧 5 个标签页中

**步骤 4：查看结果**
- 🇨🇳 **中文翻译**：原文的自然流畅中文翻译
- 📝 **语法纠错**：逐句检查语法错误，给出修改建议
- 🔍 **长难句分析**：句子结构拆解，成分标注
- 📖 **词汇拓展**：重点词汇的音标、释义、同义词、例句
- 📋 **综合分析**：融合知识库内容的全面分析

## 四、提高识别准确率的技巧

### 4.1 书写规范建议

| ✅ 推荐 | ❌ 避免 |
|---------|--------|
| 使用黑色水笔/钢笔 | 铅笔（灰度低，对比度差） |
| 在白纸上书写 | 格子纸、条纹纸、彩色纸 |
| 字迹清晰、大小适中 | 潦草连笔、字太小 |
| 保持行间距 | 行与行之间太挤 |
| 字母写完整 | 缺笔画、不闭合 |

### 4.2 拍照/扫描建议

- 📷 **光线充足**：自然光或台灯从正面照射
- 📐 **保持水平**：手机与纸面平行，避免透视变形
- 🔲 **充满画面**：文字区域占图片的 60% 以上
- 🧹 **背景干净**：避免杂乱的桌面背景

### 4.3 预处理参数调优

如果识别效果不理想，可以修改 `config.py` 中的预处理参数：

```python
# 打开 config.py，找到以下参数

PREPROCESSING = {
    "denoise_strength": 15,      # 增大去噪强度（图片噪点多时）
    "clahe_clip_limit": 3.0,     # 增大对比度增强（字迹太浅时）
    "binary_block_size": 21,     # 调整二值化粒度（字迹太粗时增大）
}
```

## 五、常见问题排查

### 问题 1：OCR 识别不到文字

**可能原因与解决：**
1. 图片质量太差 → 确保光线充足，重新拍照
2. 文字太小 → 拉近拍摄距离
3. 背景干扰 → 使用白纸，避免复杂背景
4. 置信度阈值过高 → 降低 `config.py` 中的 `OCR_CONFIDENCE_THRESHOLD` 到 0.2

### 问题 2：识别结果乱码

**可能原因与解决：**
1. 图片倾斜严重 → 确保「倾斜矫正」已开启
2. 手写太潦草 → 书写时放慢速度，字母写完整
3. 非英文内容 → 当前版本仅支持英文识别

### 问题 3：大模型分析失败

**可能原因与解决：**
1. 未配置 API Key → 按照第二章配置
2. API Key 过期或余额不足 → 登录提供商后台查看
3. 网络不通 → 检查是否能访问对应 API 地址
4. 切换提供商 → 在设置中切换到其他可用提供商

### 问题 4：知识库检索无结果

**可能原因与解决：**
1. 识别文本含噪 → 手动编辑修正识别文本后重试
2. 词汇不在库中 → 可以自行扩充 knowledge/vocabulary.json
3. 查询词与库内词不匹配 → 这是关键词匹配的正常情况

### 问题 5：启动时报错

```bash
# 常见错误及解决

# ModuleNotFoundError: No module named 'xxx'
pip install xxx

# 端口被占用
# 修改 config.py 中的 GRADIO_SERVER_PORT = 7861

# CUDA 相关错误
# 系统会自动回退到 CPU，不影响使用
```

## 六、线上部署（Hugging Face Spaces）

### 6.1 为什么要部署？

- 生成一个**公开访问链接**，可以分享给老师/同学
- 不需要自己电脑一直开机
- 满足课程设计要求

### 6.2 部署步骤（15 分钟）

**步骤 1：注册 Hugging Face**

访问 https://huggingface.co/join ，用邮箱注册

**步骤 2：创建 Space**

1. 登录后，点击右上角头像 → **New Space**
2. 填写信息：
   - Space Name: `english-ocr-assistant`
   - License: MIT
   - SDK: **Gradio**
   - Hardware: **CPU basic**（免费）
   - Visibility: **Public**
3. 点击 **Create Space**

**步骤 3：设置 API Key 密钥**

1. 进入 Space 页面 → Settings 标签
2. 找到 **Repository Secrets**
3. 添加密钥：
   - Name: `LLM_API_KEY`
   - Value: 你的 API Key
4. 保存

**步骤 4：推送代码**

```bash
# 在项目目录下初始化 Git
cd "C:\Users\lenovo\Desktop\English OCR"
git init
git add .
git commit -m "Initial commit"

# 关联 Hugging Face 仓库
git remote add space https://huggingface.co/spaces/你的用户名/english-ocr-assistant

# 推送
git push space main
```

**步骤 5：等待部署**

- 首次部署约 5-10 分钟（自动安装依赖、下载模型）
- 在 Space 页面可以看到构建日志
- 状态变为 **Running** 即部署成功

**步骤 6：获取链接**

公开访问链接格式：
```
https://huggingface.co/spaces/你的用户名/english-ocr-assistant
```

> 📌 **链接有效期**：永久有效。免费版在 48 小时无访问后会休眠，有人访问时自动唤醒。

### 6.3 部署前修改

部署到 Hugging Face 前，需要修改 `config.py` 中的知识库路径：

```python
# config.py 第 58 行附近
KB_DIR = Path("/tmp/knowledge")  # 改为 /tmp 目录
```

## 七、系统优化与自定义

### 7.1 扩充知识库

**添加新词汇：**
```json
// 编辑 knowledge/vocabulary.json
// 在 "entries" 数组中追加：
{
    "word": "ubiquitous",
    "phonetic": "/juːˈbɪkwɪtəs/",
    "meaning": "adj. 无处不在的；普遍存在的",
    "level": "考研",
    "synonyms": "omnipresent, pervasive, universal",
    "collocations": "ubiquitous presence 无处不在",
    "example": "Smartphones have become ubiquitous in modern society."
}
```
然后运行 `python knowledge/build_kb.py` 重建索引。

**批量导入：**
```bash
python -c "
from modules.knowledge_base import KnowledgeBase
kb = KnowledgeBase()
kb.load_data()
kb.batch_import('your_file.json')
"
```

### 7.2 优化提示词

编辑 `modules/prompts.py`，在 `SYSTEM_PROMPTS` 字典中修改对应任务的提示词。

例如让翻译更学术化：
```python
"chinese_translation": """\
你是一位专业的英中学术翻译专家。
翻译要求：
1. 使用规范的学术书面语
2. 保留原文的专业术语
...
"""
```

### 7.3 开启 FAISS 语义检索

如果你的网络可以访问 HuggingFace（或配置了镜像），可以启用更强的语义检索：

```bash
# 方式一：使用镜像
set HF_ENDPOINT=https://hf-mirror.com
python knowledge/build_kb.py

# 方式二：手动下载模型后离线使用
# 1. 在能上网的电脑上下载模型
# 2. 复制到本机 HF 缓存目录
# 3. 运行 python knowledge/build_kb.py
```

### 7.4 调整预处理参数

针对不同书写风格，可以调整预处理参数：

```python
# config.py

# 铅笔字迹（对比度低）：
"clahe_clip_limit": 4.0,

# 细笔迹：
"binary_block_size": 11,

# 粗笔迹：
"binary_block_size": 21,

# 严重噪点图片：
"denoise_strength": 20,
```
